# HTML Prototype Project Constitution

## Purpose
Build a high-fidelity HTML prototype that reproduces approved Figma designs for usability testing.

## Source Hierarchy
1. Project configuration
2. Screen specification
3. Figma screen
4. Pattern library
5. Existing components
6. Assets
7. Design tokens
8. Explicit user instruction

## Rules
- MUST implement one screen at a time.
- MUST NOT invent visual design, routing, copy, assets or interactions.
- MUST stop and produce a Missing Information Report if required information is absent.
- MUST reuse existing components before creating new ones.
- MUST use semantic design tokens.
- MUST report confidence below 100%.

## Required Output
- Files created
- Files modified
- Verification summary
- Visual QA
- Missing Information Report
- Confidence assessment
